Enduro

Modifie l'affichage des cartes offline (vectorielle) pour l'utilisation en moto tout terrain avec les chemins visuellement classifiés par "difficulté" liée au type d'engin et aux précisions présentes dans OpenStreetMap, permet des calculs de routage "tout terrain" pour 3 types de motos (de la plus lourde à la plus légère).

( le processus d'installation créé 3 nouveaux profils avec leur configuration et leur guidage, les profils et la configuration Osmand existante ne sont pas modifiés )

un didactitiel en vidéo https://youtu.be/07nj4LoACdc



 

Caractéristiques notables :


  • Chemins identifiés visuellement par carrossabilité (couleurs) explications

  • POI utiles / intéressants mis en évidence

  • Difficulté des sentiers à partir des tags VTT et pédestre explications

  • Simplification pour lisibilité des itinéraires VTT/pédestre

  • Réglages supplémentaires pour la lisibilité en roulant

  • Fonctionnement du style

  • Fonctionnement des routages

Instructions d'installation


  • Télécharger ce fichier

Ce fichier contient :

  • 1 rendu
  • 3 profils (Enduro, Grand trail, Petit trail)
  • 3 routages (Enduro, Grand trail, Petit trail)

Première installation : Mise à jour du rendu / Routage :
  • Télécharger le fichier
  • Ouvrir avec OsmAnd
  • Cocher toutes les cases pour importer tous les fichiers
  • Terminé !

    Les profils avec rendu + routage sont prédéfinis
  • Télécharger le fichier
  • Ouvrir avec OsmAnd
  • Cocher uniquement les cases "ressources" pour n'importer que les fichiers de rendu et de routage
  • Cliquez sur "Tout remplacer"

  • Terminé !

    Le nouveaux rendu et les routages ont remplacé les anciens fichiers


 

Documentation


fonctionnement


le style d'affichage est appliqué automatiquement aux 3 profils créés lors de l'installation.


ces 3 profils sont identiques à part la couleur et le routage qui leur est affecté par le processus d'installation.
ces 3 profils sont nécessaires pour utiliser les 3 routages différents fournis.

  • leurs options proviennent du fichier de style et du fichier de routage qui leur est affecté
  • les options du style sont regroupées à la fin de l'onglet "Paramétrer la carte" dans le sous menu "Autres caractéristiques de la carte"


Options
cacher balisage enlève l'affichage des balisages à l'écran (surlignage et icône)
cacher chemin sans info passe les chemins sans précisions OpenStreetMap en trait continu
moins détaillé bascule vers le mode routier
en cas de carte arrière plan ajoute un sous lignage aux chemins pour mieux les voir par dessus des photos
cacher difficulté masque l'affichage des points colorés sur la carte
comme IGN permet d'aligner la représentation des chemins sur celle de l' IGN lorsque les cartes sont en arrière plan
Accès restreints désactive l'affichage des interdictions et reserve/parc national
niveau de gris jeu de couleur alternatif pour la couverture végétale
cacher commentaire masque les commentaires OpenStreetMap des chemins sur la carte

Si les options apparaissent "en vrac" dans le menu,
aller dans "Configurer le profil" > "Personnalisation de l'interface utilisateur" > "Paramétrer la carte" et tout en bas "Rétablir les paramètres par défaut" vous permettra de retrouver un menu bien rangé dans lequel vous pouvez replier les sous catégories.



le style mets en avant les chemins / sentiers tel que défini ci-dessous

les chemins
les sentiers


les chemins sont :

  • en trait vert continu si carrossable selon la météo
  • sinon vert pointillés pour les chemins d'exploitation (chemin de champs en langage courant)
  • les pistes empierrées carrossables sont en marron continu ou pointillé selon leur qualité
  • les sentiers sont en pointillés noir

  • un switch "cacher chemin sans info" dans les options du style permet de basculer les chemins sans précisions dans OSM en trait continu, les pistes restent en marron et les sentiers en noir.

mode "comme IGN"

dans les 3 profils il est possible de basculer l'affichage dans un mode "comme IGN" depuis les options du style pour

  • d'une part ressembler aux cartes IGN 1/25000 bleue (meilleure lisibilité de certains éléments au détriment de précisions sur les chemins)
  • d'autre part permettre l'affichage en "sous couche" des cartes IGN fournie en "live" ou préalablement enregistrées sans perturber la lisibilité.

lors de l'utilisation du mode "comme IGN" avec les cartes en arrière plan les éléments suivants sont désactivés

  • ruisseau / rivière
  • points au sol sauf barrières
  • textes saufs villes/villages
  • courbes de niveau

les niveaux d'apparition des éléments sont synchronisés avec celui des cartes.


mode "moins détaillé"

pour une utilisation "routière" (idéalement en mode "nuit")

  • seules les pistes sont représentées en gris vert
  • pas de balisages
  • moins de points (nature / sol etc)


mode "en cas de carte arrière plan"

disponible dans les 3 profils

  • ajoute un "sous lignage" sombre aux chemins pour les rendre plus visible sur imagerie satellitaire
  • bascule les sentiers en gris clair pour la lisibilité
  • couleurs adaptées pour divers éléments


(la lisibilité est améliorée pour les texte en basculant sur le mode nuit )


mode "nuit"

pour économiser de la batterie avec un écran OLED et obtenir une lecture contrastée simplifiée lors du roulage.

  • adapte les couleurs de divers éléments
  • désactive certains éléments



 

Routage


Calcul d'itinéraire


Des calculs de routage pour 3 types d'utilisation moto, chacun de ces routages calcule un itinéraire en fonction des infos disponibles dans OpenStreetMap.


voir ici pour l'installation
explications sur les catégories de chemins


Gros trail

ne vous fera passer QUE par les chemins faciles (les marrons) et s'il n'en trouve pas, par la route et de tout petits bouts de certains chemins verts.

  • respecte les interdictions (chemins en rouge sur la carte)
  • respecte les barrières (points rouges sur la carte)
  • les chemins "privés" sont gérés comme avec le profil par défaut d' OsmAnd.
  • les chemins "interdits" sont regroupés sous le switch "pas d'interdiction" dans "Eviter les routes".
  • :bulb: il y-à un autre switch dans "Eviter les routes" pour activer un mode "pas de chemins" qui évite au maximum les chemins en restant sur les routes les plus petites possible.
  • :bulb: il y-à un autre switch dans "Eviter les routes" pour activer un mode "si chemins secs" qui rajoute quelques chemins "verts" de la carte au cas ou il n'y aurai pas de chemins "marrons" à proximité.
    :warning: Attention à la boue avec des pneus non adaptés !
  • considère toutes les routes au même niveau et donc tracera au plus direct.
  • :bulb: il y-à 2 autres switchs dans "Eviter les routes" pour activer un mode "moins de route" et "encore moins de route" qui permettent l'un ou l'autre de trouver plus de chemins si vous faites un trajet de moins de 200 km pour le premier et 100 km pour le second.
    :warning: le temps de calcul est augmenté et le risque de plantage de l'appli aussi !

(mettez des points intermédiaires pour "forcer" le détour par les "bons" chemins s'il n'y en à pas en ligne directe).


Petit trail

vous fera éviter les sentiers et footway mais prendra tous les chemins sauf ceux indiqués "impassable"

  • respecte les interdictions (chemins en rouge sur la carte)
  • respecte les barrières (points rouges sur la carte)
  • les chemins "privés" sont gérés comme avec le profil par défaut d' OsmAnd.
  • les chemins "interdits" sont regroupés sous le switch "pas d'interdiction" dans "Eviter les routes".
  • dissuade d'empreinter les chemins trop difficiles (vous pouvez poser un point de passage dessus pour les prendre délibéremment).
  • autorise le passage sur les sentiers "faciles" points verts sur la carte.
  • considère tous les chemins de la même façon (pas de carrossable ou de boueux)
    :bulb: SAUF si le switch "trie les chemins" dans "Eviter les routes" est actif (coché) il va hiérarchiser les chemins du plus carrossable à inconnu.
    dans les faits il va pénaliser de plus en plus les chemins qui sont les moins praticables
    pour aller d'un point A à un point B si 2 chemins à peu près équivalents en distance existent, le moteur de calcul du routage prendra le plus praticable, la proportion de route augmente si les chemins ne sont pas renseignés dans OpenStreetMap.
  • considère toutes les routes au même niveau et donc tracera au plus direct.
  • :bulb: il y-à un autre switch dans "Eviter les routes" pour activer un mode "moins de route" qui permet de trouver plus de chemins si vous faites un trajet de moins de 200 km .
    :warning: le temps de calcul est augmenté et le risque de plantage de l'appli aussi !

( ne mettez pas de points trop éloignés pour accélérer le calcul -tous les 100 km par exemple )


Enduro

vous fera passer partout sauf pistes cyclables et bien sur les points noirs (impassables ^^).

  • tous les switchs sont inactifs (tout est actif, les gués, les routes gelées, l'accès au privé etc)
  • respecte les interdictions "no" et "private" SAUF si le switch "Pas d'interdit" est actif.
  • respecte les barrières absolues (chaine, portail etc)
  • priorité inférieure pour les points rouges de la carte qui seront évités si alternative.
  • considère les footway comme des path (beaucoup de sentiers sont indiqués footway à tort)
  • privilégie les itinéraires balisés (rando et VTT)
  • privilégie légèrement les sentiers aux chemins.
  • toutes les routes sont au même niveau il trace au plus direct.

( ne mettez pas de points trop éloignés pour accélérer le calcul - tous les 100 km par exemple )
chez moi un calcul entre 2 points distants d environ 200 km prend 2 minutes.


Calcul

Si vous trouvez le calcul trop lent ou que vous n'arrivez pas à calculer un itinéraire

  • aller dans "configurer le profil"
  • puis "parametres de guidage"
  • "caractéristiques du véhicule"
  • "vitesse par defaut"

et baissez la vitesse max vers 90 (ou moins si besoin)
sachez que plus vous la baissez, plus le calcul est rapide mais plus il empreinte de routes.
(n'interdisez pas les autoroutes ou péages, ils sont déjà désactivés)
